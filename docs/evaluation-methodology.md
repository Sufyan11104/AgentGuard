# Evaluation Methodology

AgentGuard uses deterministic synthetic tests to evaluate known AI-agent failure modes.

## Scoring

Each test case produces one result:

- `passed`: deterministic checks passed.
- `open`: a deterministic failure was observed.
- `needs_review`: no deterministic failure, but the result needs human review.

Run summaries include total counts, severity counts, and a 0-100 score.

## Severity Weighting

Severity is stored on each test case as:

- `low`
- `medium`
- `high`
- `critical`

Severity counts help reviewers prioritize remediation. The current score model is intentionally simple and deterministic, making it easy to understand and test.

## Assertion Types

Supported assertion types include:

- `must_not_contain`
- `must_contain`
- `should_refuse`
- `must_not_call_tool`
- `requires_human_approval`
- `must_not_claim_action_completed`

Assertions are evaluated against the target response text and dry-run tool-call evidence.

## Why Tests Are Deterministic

AgentGuard is designed for repeatable local evaluation. Deterministic checks make it suitable for CI, demos, and regression testing. Human review remains important for ambiguous behavior.

## Report Generation

Reports are generated by `@agentguard/core`:

- JSON for automation.
- Markdown for code review and tickets.
- HTML for local demos and sharing.

`@agentguard/db` can persist all three formats for dashboard display.

## Limitations

- Deterministic checks do not cover every model behavior.
- Synthetic tests are not a substitute for a full red-team exercise.
- Provider-specific model behavior can vary.
- Passing results should be treated as evidence, not a guarantee.
